# Research Scan — 2026-04-11

## Scope
Post-hardening scan: security, i18n, type safety, error handling, UX dead-ends, and silent failures all completed. This report covers remaining gaps found after those passes.

## Findings

### REMOVED — Dead Code
- `courseConceptsAtLevel2OrAbove: number` in `MasteryStats` interface (`mastery-v2.ts:20`)
  — declared but never read by `evaluateLevel()`. Initialized to `0` in `attempts/route.ts:169`.
  — **Action taken**: removed from interface and call site.

### FIXED — Accessibility
- `ShareButton`: copy and share icon-only buttons had `title` but no `aria-label`. Screen readers
  announce button purpose via `aria-label`, not `title`. **Fixed**: added `aria-label` matching `title`.
- `VoiceNotesPanel` clarification mic button: icon-only, no `title`, no `aria-label`.
  **Fixed**: added `aria-label={t("voiceNotes.talkNote")}` (reuses existing key).

### FALSE POSITIVE — ChunkLesson submit
- Research suggested adding loading state to `handleSubmitCheckpoint`. On inspection: the function
  is fully synchronous (local state update only). No network call, no spinner needed.

## Remaining Known Gaps
- **Vercel env vars** (UPSTASH_REDIS_REST_URL, UPSTASH_REDIS_REST_TOKEN, SUPABASE_SERVICE_ROLE_KEY) — blocked on user
- **OutlineTree keyboard navigation** — no keyboard support for expand/collapse/add/delete
- **Mobile viewport testing** — not formally tested on narrow screens

## Assessment
App is production-ready. No critical or high-severity issues remaining. Remaining items are
enhancement-level (keyboard nav, mobile polish).
