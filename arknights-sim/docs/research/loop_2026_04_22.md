## 1. Batch-Generating No-Skill Operator Stub Files (1★/2★)

**Context:** 1★/2★ operators have no skills — their "test" is just: instantiate, assert stats within expected range, assert `skill is None`. Today's pattern requires one `data/characters/X.py` + one `tests/test_v2_X.py` per operator.

**Faster patterns:**

- **`gen_characters.py --rarity TIER_1 TIER_2`** already works — it writes `data/characters/generated/X.py` stubs. The only missing piece is test coverage.
- **Single parametrized test file for all no-skill operators:** Create `tests/test_v2_no_skill_operators.py` that imports a list of `(make_fn, expected_stats)` tuples and runs one `test_instantiation[name]` per operator. Zero per-operator boilerplate — adding a new 1★/2★ means appending one dict entry to the list.
- **Script: `tools/gen_no_skill_tests.py`** — reads `data/characters/generated/` for stubs with no skill constants, emits the operator table entry. Could be a 40-line script that produces the full parametrized list.
- **Pattern already proven inside the project:** `test_v2_beagle_s1.py` shows that `make_X()` functions are the only import needed; no World setup required for stat assertions. No-skill tests need even less.

**Concrete action:** One 50-line `tests/test_v2_stubs.py` with `@pytest.mark.parametrize("factory,name", [...])` covers all no-skill operators. Adding an operator = adding one tuple. No new files.

## 2. Pytest conftest.py Fixture Sharing — Eliminating 400-File Boilerplate

**Current state:** 368 of 385 test files define identical `_world()` (6×3 GROUND grid, `dp_gain_rate=0`, `register_default_systems`). That function is copy-pasted 368 times.

**Best practices:**

- **Named fixture in `tests/conftest.py`:** Convert `_world()` to a `world` pytest fixture. Tests declare `def test_foo(world):` — no import, no boilerplate. Pytest injects it automatically because `tests/conftest.py` is discovered.
  ```python
  # tests/conftest.py  (add to existing file)
  import pytest
  @pytest.fixture
  def world():
      grid = TileGrid(width=6, height=3)
      for x in range(6):
          for y in range(3):
              grid.set_tile(TileState(x=x, y=y, type=TileType.GROUND))
      w = World(tile_grid=grid)
      w.global_state.dp_gain_rate = 0.0
      register_default_systems(w)
      return w
  ```
- **Parametrized world sizes:** Add a `world_10x5` fixture for the 14 tests using 10×5 grids. Two fixtures cover 100% of current usage.
- **`slug` and `deploy_op` fixtures:** The `_slug()` and `_deploy_X()` helpers are also near-identical across files. A `slug_factory` fixture (returns a callable with defaults) eliminates that too.
- **Migration cost:** Low. Old tests that call `_world()` still work — just change the function name to use the fixture argument. No logic changes needed. A sed one-liner renames `_world()` calls to `world` across all files: `sed -i '' 's/w = _world()/w = world/g'`.
- **Scope:** Keep fixtures at default (`function` scope) — world state is mutated per test, so sharing across tests would cause interference.

**Do NOT do:** Session-scoped world fixture — combat state is mutated; sharing across tests produces flaky results.

**Yield:** Eliminating 368 × 10-line `_world()` copies saves ~3,680 lines. New operator tests become 15 lines instead of 40.
