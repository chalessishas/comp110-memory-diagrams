## Research Question 1: TOEFL 2026 Format Changes

### Confirmed New Format (Effective January 21, 2026)

The TOEFL iBT writing section was completely restructured. The old Integrated + Independent 50-minute essay pair is gone.

**New 3-task structure (~23 minutes total):**

| Task | Time | Min Words | Scoring |
|------|------|-----------|---------|
| Build a Sentence | ~6 min | N/A | Binary correct/incorrect |
| Write an Email | 7 min | 80–120 words | 0–5 rubric |
| Write for an Academic Discussion | 10 min | 100–150 words | 0–5 rubric |

**Key changes relevant to our scorer:**

1. **Academic Discussion minimum is 100 words** (not 120). Our current `WORD_COUNT_TARGETS.discussion.min = 80` is fine (conservative), and `target = 130` is within the 100–150 band. No change needed.

2. **Email minimum is 80–120 words.** Our `email: { min: 50, target: 120, max: 200 }` — the min=50 is too lenient; raising to 70 would be more accurate. Low priority.

3. **Scoring scale is 0–5 in 0.5 increments.** Our scale matches.

4. **Build-a-Sentence is binary.** Already handled: `if (taskType === 'build-a-sentence') return { overall: 0, ... }` — correct skip.

5. **No rubric text changes found** for Academic Discussion Score 4 vs 5 criteria. The ETS rubric PDF confirms the same language: Score 5 = "precise, idiomatic word choice + well-elaborated"; Score 4 = "appropriate word choice + adequately elaborated". The format changed but rubric language is unchanged.

**Conclusion:** No urgent scorer changes required from format update. Email word count floor could be raised from 50→70 (cosmetic, low priority).

### What's Already Implemented

- AWL tiered (sublists 1–3 basic @ 0.015/word cap 0.12; sublists 4–10 advanced @ 0.04/word cap 0.08)

### New Features Found — Implementable in Pure JS

#### F1: Academic Phrasal Verb Density (HIGH VALUE — not yet implemented)

**Research basis:** Eguchi & Kyle (2023), Cambridge Core SSLA studies on L2 writing: collocate frequency and collocational accuracy are rated as distinct from vocabulary diversity. The transition from Score 4 → Score 5 is characterized by "particles, compounds, and fixed phrases" — the exact category AWL does NOT capture.

ETS rubric language: "precise, **idiomatic** word choice" (Score 5) vs. "appropriate word choice" (Score 4). Idiomaticity = correct phrasal verb and preposition collocation, which Chinese L1 writers systematically avoid (they default to Latinate single-word synonyms).

**What to detect:** Presence of natural English academic phrasal collocations that Chinese L1 writers avoid.

```js
// vocabulary.js — phrasal verb idiomaticity bonus
// Research: Nesselhauf 2003; Eguchi & Kyle 2023; ETS Score-5 rubric criterion
// These are unambiguous — low false-positive risk
const ACADEMIC_PHRASAL = [
  /\bpoint(s|ed|ing)?\s+out\b/i,        // "points out" vs formal "indicates"
  /\baccount(s|ed|ing)?\s+for\b/i,      // "accounts for" — precise causation
  /\bbrought?\s+about\b|\bbring(s|ing)?\s+about\b/i,
  /\bresult(s|ed|ing)?\s+in\b/i,
  /\btook?\s+into\s+account\b|\btaking\s+into\s+account\b/i,
  /\bcarr(y|ies|ied|ying)\s+out\b/i,
  /\bcall(s|ed|ing)?\s+for\b/i,         // "calls for" = requires/demands
  /\bbuilt?\s+on\b|\bbuild(s|ing)?\s+on\b/i,
  /\bdraw(s|n|ing)?\s+on\b/i,           // "draws on research"
  /\bset(s|ting)?\s+(out|aside|up)\b/i,
  /\bstand(s|ing)?\s+out\b/i,
  /\bfocus(es|ed|ing)?\s+on\b/i,        // very common, intentionally included
]
const phrasalHits = ACADEMIC_PHRASAL.filter(re => re.test(text)).length
const phrasalBonus = phrasalHits >= 4 ? 0.06 : phrasalHits >= 2 ? 0.03 : 0
```

**Estimated QWK gain:** 0.04–0.06 (addresses Score 4→5 gap directly)
**False positive risk:** LOW — these patterns are unambiguous and productive in academic writing
**Where to add:** `vocabulary.js`, added to the final value formula

#### F2: Lexical Sophistication via Low-Frequency Academic Bigrams (MEDIUM VALUE)

**Research basis:** Biber et al. (1999) "Longman Grammar of Spoken and Written English"; Kyle & Crossley (2015) on L2 writing — multi-word unit density predicts holistic scores independently of single-word AWL.

**What to detect:** Certain academic 2–3 word phrases that appear frequently in high-scoring L2 essays but are absent in lower-scoring ones. These are formulaic academic sequences not captured by AWL:

```js
const ACADEMIC_BIGRAMS = [
  'as a result', 'in particular', 'on the other hand', 'in terms of',
  'in contrast to', 'in response to', 'as opposed to', 'with regard to',
  'in light of', 'in line with', 'play a role', 'a significant role',
  'take into consideration', 'give rise to', 'give an example',
]
const bigramHits = ACADEMIC_BIGRAMS.filter(b => lower.includes(b)).length
const bigramBonus = bigramHits >= 4 ? 0.04 : bigramHits >= 2 ? 0.02 : 0
```

**Assessment:** Several of these bigrams overlap with existing `MARKER_CATEGORIES` in organization.js (e.g., "as a result", "on the other hand", "in particular"). Must audit overlap before adding — risk of double-counting the same signal.

**Estimated QWK gain:** 0.02–0.03 net (after deduplication)
**False positive risk:** LOW
**Recommendation:** Implement only if phrasal verb bonus (F1) alone proves insufficient. F1 is cleaner.

### New Patterns Found — Implementable Without NLP Parser

#### G1: Tense Inconsistency Detection (HIGH VALUE — not yet implemented)

**Research basis:** Tsai (2023) Sage Journals — tense/aspect shift ranked #2 error in Chinese-English translation. Li (2024) Language Learning: "Chinese learners are conceptually blind to temporal differences conveyed by tense." TOEFL academic discussion is typically written in present tense; essays that shift inconsistently to past tense mid-paragraph signal L1 interference.

**Error pattern:** Essay is primarily present tense, then flips to past tense with no discourse reason (not citing past events).

**Implementation strategy:** Count present-tense verbs vs. past-tense verbs in the essay. If past tense is > 20% of clause verbs AND no time-anchoring words are present (yesterday, last year, in [year]), flag as possible inconsistency.

```js
// grammar.js addition — tense consistency proxy
// Counts surface present vs. past forms. Not full parse — approximation only.
// Only flag if there's a significant mix with no temporal anchors.
const presentVerbRe = /\b(is|are|has|have|do|does|seems?|shows?|suggests?|indicates?|means?|affects?|helps?|works?|requires?|provides?)\b/gi
const pastVerbRe    = /\b(was|were|had|did|seemed|showed|suggested|indicated|meant|affected|helped|worked|required|provided)\b/gi
const temporalAnchorRe = /\b(yesterday|last (year|month|week)|in (19|20)\d{2}|ago|previously|recently)\b/i

const presentCount = (text.match(presentVerbRe) || []).length
const pastCount    = (text.match(pastVerbRe) || []).length
const total = presentCount + pastCount
if (total >= 8 && pastCount / total > 0.30 && !temporalAnchorRe.test(text)) {
  errors.push(`Possible tense inconsistency: ${pastCount} past-tense forms mixed with ${presentCount} present-tense forms without clear temporal reference`)
}
```

**Estimated QWK gain:** 0.03–0.05 (tense errors are frequent and penalized by e-rater)
**False positive risk:** MEDIUM — must tune the 0.30 threshold carefully. Essays that legitimately discuss past events or use past in examples would trigger false positives. Temporal anchor exclusion mitigates but doesn't eliminate.
**Recommendation:** Implement with threshold 0.35 (not 0.30) to stay conservative. Add to `grammar.js` errors list.

#### G2: Past Tense Regularization Errors (MEDIUM VALUE — partially captured)

**Research basis:** Chinese has no verb morphology; learners omit past -ed or use regularized forms for irregular verbs.

**Current coverage:** The `THIRD_PERSON_BARE` and `HAVE_BEEN_BARE` patterns catch some of this, but only for specific verbs in 3P context.

**Gap:** Irregular verb regularization — "goed", "writed", "buyed", "thinked" — is not currently checked.

```js
// grammar.js — irregular verb regularization errors (Chinese L1 transfer)
const IRREGULAR_REGULARIZED = [
  { re: /\bgoed\b/i, msg: 'Verb error: "goed" → "went"' },
  { re: /\bwrited\b/i, msg: 'Verb error: "writed" → "wrote"' },
  { re: /\bbuyed\b/i, msg: 'Verb error: "buyed" → "bought"' },
  { re: /\bthought?ed\b/i, msg: 'Verb error: "thinked/thoughted" → "thought"' },
  { re: /\bteached\b/i, msg: 'Verb error: "teached" → "taught"' },
  { re: /\bspeaked\b/i, msg: 'Verb error: "speaked" → "spoke"' },
  { re: /\bfeed\b.{0,5}\bpast\b|\bfeeded\b/i, msg: 'Verb error: "feeded" → "fed"' },
  { re: /\bchoosed\b/i, msg: 'Verb error: "choosed" → "chose"' },
  { re: /\bthinked\b/i, msg: 'Verb error: "thinked" → "thought"' },
]
```

**Assessment:** These are very rare in intermediate TOEFL writers (who mostly know common irregular verbs). TOEFL test-takers are not beginners. These errors will almost never fire.

**Estimated QWK gain:** <0.01 (too rare to matter)
**Recommendation:** NOT worth implementing. The errors are so rare in TOEFL band that they wouldn't affect calibration.

#### G3: Missing Relative Clause Verb (Chinese L1 Rank 7)

**Research basis:** Chinese relative clauses are pre-nominal and verb-less; transfer produces: "The student who smart..." → "The student who is smart...". Pattern is detectable with a regex.

```js
// Detects: who/which/that followed immediately by an adjective (copula omission in relative clause)
const REL_CLAUSE_COPULA = /\b(who|which|that)\s+(very\s+)?([A-Z]?[a-z]{3,})\s+\w+/gi
// Only fire if the word after who/which/that looks like an adjective (ends in -ful, -less, -ous, -al, -ive, -ing excluded)
```

**Assessment:** Too prone to false positives without a POS tagger. "The idea that supports X" would trigger unless carefully excluded. "who/which/that" relative clauses with transitive verbs are extremely common and correct.

**Estimated QWK gain:** 0.02 if implemented perfectly; high FP risk makes net gain potentially negative.
**Recommendation:** NOT worth implementing without a parser. Skip.

## Research Question 4: Score 4 vs Score 5 Discriminators

### Official Rubric Difference (Confirmed from ETS PDF + Writing30)

| Criterion | Score 4 | Score 5 |
|-----------|---------|---------|
| Elaboration | Adequately elaborated | **Well**-elaborated |
| Word choice | Appropriate | **Precise, idiomatic** |
| Errors | Few | Almost none |
| Syntax | Variety of structures | **Effective use** of variety |

### Concrete Linguistic Discriminators Not Yet Implemented

#### D1: Sentence-Initial Connector Placement (HIGH VALUE)

**Research basis:** Ferris (1994) discourse analysis of ESL academic writing; Crossley et al. (2020) TAACO: connector **position** (sentence-initial vs. mid-sentence) is a stronger predictor of holistic score than connector frequency. Score-5 writers open body paragraphs with transition words; Score-4 writers bury them mid-sentence.

**Current gap:** `organization.js` counts connector presence but not position.

**Implementation:**
```js
// organization.js — paragraph-initial connector bonus
// Reward: ≥2 sentences starting with inferential/contrast marker → +0.06
const sentences = text.split(/[.!?]+/).map(s => s.trim()).filter(Boolean)
const PARA_INIT_MARKERS = /^(However|Furthermore|Moreover|Therefore|Consequently|Nevertheless|In contrast|By contrast|Admittedly|Additionally|Notably|Significantly)\b/i
const paraInitCount = sentences.filter(s => PARA_INIT_MARKERS.test(s)).length
const paraInitBonus = paraInitCount >= 3 ? 0.06 : paraInitCount >= 2 ? 0.04 : paraInitCount === 1 ? 0.02 : 0
```

**Estimated QWK gain:** 0.04–0.06
**False positive risk:** LOW — sentence-initial connector is always good style; this never fires incorrectly
**Where to add:** `organization.js`, added to final value formula (already noted in prior loop9 draft)

#### D2: Mean Sentence Length Sweet Spot (MEDIUM VALUE)

**Research basis:** Lu (2010, 2012) SCA studies: Mean Length of T-unit (MLT) is one of top-3 syntactic complexity features used in AES validation. Score-5 TOEFL essays average 14–18 words/sentence. Score-3 averages 10–13. Our style.js only measures **variance**, not mean length.

**Current gap:** style.js measures variance but not average sentence length.

```js
// style.js — mean sentence length bonus
const sentences = text.split(/[.!?]+/).map(s => s.trim()).filter(Boolean)
const totalWords = tokens.length
const meanSentLen = sentences.length > 0 ? totalWords / sentences.length : 0
// Sweet spot: 13-22 words/sentence → +0.04
// Short (< 10): choppy, underdeveloped → 0
// Very long (> 28): run-on risk → 0
let sentLenBonus = 0
if (meanSentLen >= 13 && meanSentLen <= 22) sentLenBonus = 0.04
else if (meanSentLen >= 11) sentLenBonus = 0.02
```

**Estimated QWK gain:** 0.03–0.04
**False positive risk:** LOW — run-ons already checked in grammar.js; this is independent
**Where to add:** `style.js`, added to final value formula (was in prior loop9 draft but not implemented)

#### D3: Elaboration Depth via Subordinate Clause Density (NEW — not in prior research)

**Research basis:** From ETS Academic Discussion guide and toeflresources.com: "You must make an argument and elaborate on it... 120 words + one subordinating conjunction + three coordinating conjunctions (two unique)." The exact formula for Score 5 is anchored to subordinating conjunctions. Our style.js counts subordinating conjunction patterns as part of syntactic variety (COMPLEX_PATTERNS[4]), but does not measure **density**.

**What to add:** A minimum subordinating clause density check. Score-5 writers use subordination to elaborate; Score-4 writers may use simple sentences with coordinating conjunctions only.

```js
// Count subordinating conjunctions as density metric
// Thresholds based on: 100-word essay with ≥2 subordinators → well-elaborated
const subordinators = /\b(although|because|since|while|whereas|when|if|unless|after|before|once|as long as|provided that|given that|even though|even if)\b/gi
const subCount = (text.match(subordinators) || []).length
const wordCount = tokens.length || 1
const subDensity = (subCount / wordCount) * 100  // per 100 words
// 2+ subordinators/100 words is normal for Score-4+; 3+ suggests Score-5 elaboration
let subBonus = 0
if (subDensity >= 3.0) subBonus = 0.04
else if (subDensity >= 1.5) subBonus = 0.02
```

**Estimated QWK gain:** 0.03–0.05
**False positive risk:** LOW — subordinating conjunction overuse doesn't happen in TOEFL-length essays
**Where to add:** `style.js` (syntactic maturity) or `development.js` (elaboration depth)
**Caution:** style.js already captures `although|despite|while|whereas` in COMPLEX_PATTERNS[4]. Subbonus would partially double-count. Need to ensure the new feature tracks **density**, not just presence.
