## Candidate 1 (VOCABULARY): "With the development of…" Opening Formula Penalty

### Corpus Evidence
Corpus study of Chinese L1 academic essays (CCSENet ELT, 2022 — "A Contrastive Study on Lexical Bundles in Argumentative Writing"): the 4-gram "with the development of" appeared at **12.4×** the rate of native English writers in L1-Chinese TOEFL-style essays. It is a direct calque from a Chinese discourse frame (随着…的发展) used to introduce background context. Native academic writers virtually never use this exact phrase; they use constructions like "as X has grown", "given the rise of", "driven by advances in".

Same study also documented "in today's society", "in the modern world", "in recent years" as near-universal Chinese L1 essay openers, appearing at 7–9× native rate.

### Mechanism
These phrases are NOT caught by the existing `FORMULAIC_BUNDLES` list (which targets hedging/belief frames). They signal shallow background-setting, not hedging. They would produce false positives at near-zero rate for native writers because native academic writers do use "in recent years" occasionally — so penalty must only fire on **3+ total hits or 2+ from the same phrase**.

### Code Sketch (vocabulary.js)
```js
// Chinese L1 background-opening formula penalty (Loop 34 candidate)
// Corpus: CCSENet ELT 2022 — 12.4× overuse of "with the development of" in L1-Chinese essays.
// Native academic writers rarely use these exact frames (≤1 per essay on average).
// Penalty fires only at ≥2 same-phrase OR ≥3 total to suppress FP on native writers.
const BG_OPENING_FORMULAS = [
  /\bwith the development of\b/gi,
  /\bin today'?s society\b/gi,
  /\bin the modern world\b/gi,
  /\bwith the rapid development of\b/gi,
  /\bwith the advancement of\b/gi,
  /\bin recent years,?\s/gi,
]
const bgHits = BG_OPENING_FORMULAS.map(re => (text.match(re) || []).length)
const maxBgSingle = Math.max(...bgHits)
const totalBgHits = bgHits.reduce((a, b) => a + b, 0)
const bgOpeningPenalty = maxBgSingle >= 2 ? 0.04 : totalBgHits >= 3 ? 0.02 : 0
```

Then in the `value` computation:
```
- bgOpeningPenalty
```

### Frequency Estimate
- Chinese L1 Score-3 TOEFL essays: ~60–70% contain at least one of these phrases; ~30% contain 3+
- Native Score-5 essays: <5% contain any (and never more than once)
- Expected impact: -0.02 to -0.04 penalty on typical Score-3/4 Chinese L1 essays → helps close Score3→4 gap

### FP Risk
LOW. "In recent years" can legitimately appear once — threshold ≥2 of same phrase prevents single-use FP. Native writers virtually never write "with the development of" at all. Estimated FP rate: <1%.

### Priority: HIGH
Directly addresses calibration gap between Score3 and Score4. Targets a well-documented Chinese L1 transfer pattern not yet covered by any existing penalty. Implementation is 10 lines.

## Candidate 2 (STYLE): Additive/Temporal Sentence-Opener Monotony Penalty

### Corpus Evidence
Two independent corpus studies converge:
1. **ScienceDirect (2011)** "The use of cohesive devices in argumentative writing by Chinese EFL learners at different proficiency levels": Chinese EFL learners overuse additive ("and, also, besides, in addition, moreover, furthermore") and temporal ("first, first of all, secondly, thirdly, finally") connectors as sentence-initial devices.
2. **ResearchGate (2014)** "Investigating the use of cohesive devices by Chinese EFL learners": sentence-initial conjunction placement accounts for **45.4% of total conjunction uses** in Chinese L1 essays vs **17.0%** in native English speaker essays.

The specific pattern: essays where every body sentence opens with "First, ... Second, ... In addition, ... Furthermore, ... Finally," are a known low-proficiency marker. These are not wrong — but using 4+ of them as sentence openers in a single essay signals formulaic paragraph-template writing rather than developed argumentation.

### Mechanism
The existing `style.js` already penalizes `Besides` sentence-initial overuse (Loop 26) and `iOpenerPenalty`. This is the same concept extended to the full set of additive/temporal transitional openers. Crucially, the threshold must be HIGH (≥5 sentence-initial uses total) to avoid penalizing legitimate use of discourse markers.

### Code Sketch (style.js)
```js
// Additive/temporal sentence-opener monotony penalty (Loop 34 candidate)
// ScienceDirect (2011) Chinese EFL cohesion study: overuse of additive/temporal
// sentence-initial connectors at 2.7× native rate. Template essays where every body
// sentence opens with First/Second/In addition/Furthermore/Finally score lower on ETS rubric
// for "variety in sentence structure and discourse organization."
// Guard: ≥5 hits (not ≥2) — ensures legitimate use never fires.
// Email exempt: sequential discourse markers are natural in email tasks.
let transitionalMonotonyPenalty = 0
if (taskType !== 'email') {
  const SENT_INITIAL_ADDITIVES_RE = /(?:^|[.!?]\s+)(?:First(?:ly|,| of all)|Second(?:ly|,)|Third(?:ly|,)|Fourth(?:ly|,)|In addition[,\s]|Furthermore[,\s]|Moreover[,\s]|Additionally[,\s]|Also[,\s]|Lastly[,\s]|Finally[,\s])/gim
  const transitionalHits = (text.match(SENT_INITIAL_ADDITIVES_RE) || []).length
  transitionalMonotonyPenalty = transitionalHits >= 5 ? 0.04 : transitionalHits >= 4 ? 0.02 : 0
}
```

### Frequency Estimate
- Chinese L1 Score-3 five-paragraph essays: ~40–50% will have ≥4 such openers
- Native Score-5 essays: ~5–10% might hit ≥4 (well-organized essays do use transitions, but varied)
- FP risk is the main concern here — see below

### FP Risk
MEDIUM. A well-organized native essay could legitimately use First/Second/Third/In addition/Finally (5 total) without any problem. The threshold at ≥5 does NOT perfectly discriminate. 

**Mitigation**: Pair with a "variety check" — only penalize if the transitional openers use ≤3 DISTINCT types out of the 5+ hits (meaning the writer is repeating the same 1–2 openers, not using a diverse set). This nearly eliminates FP from well-organized essays.

**Revised code:**
```js
if (taskType !== 'email') {
  const SENT_INITIAL_ADDITIVES_RE = /(?:^|[.!?]\s+)(First(?:ly|,| of all)|Second(?:ly|,)|Third(?:ly|,)|Fourth(?:ly|,)|In addition[,\s]|Furthermore[,\s]|Moreover[,\s]|Additionally[,\s]|Also[,\s]|Lastly[,\s]|Finally[,\s])/gim
  const transitionalMatches = [...text.matchAll(SENT_INITIAL_ADDITIVES_RE)].map(m => m[1].replace(/[,\s].*$/, '').toLowerCase())
  const distinctTypes = new Set(transitionalMatches).size
  const totalHits = transitionalMatches.length
  // Only penalize when openers are high-count AND low-variety (template writing)
  const isMonotone = totalHits >= 4 && distinctTypes <= 2
  transitionalMonotonyPenalty = isMonotone ? 0.03 : 0
}
```

### Priority: MEDIUM
Addresses a real Chinese L1 style gap but needs careful threshold tuning. Lower priority than Candidate 1 because the FP risk is non-trivial. Recommend implementing AFTER Candidate 1.

## Candidate 3 (VOCABULARY): Academic Hedging Overuse Penalty Expansion

### Corpus Evidence
**Frontiers AI 2023** (lexical complexity features): "It is important to note", "it should be noted", "it is worth noting", "it is essential to" are classified as "stance markers" and appear at **2.8× normal frequency** in lower-proficiency L2 essays. Research distinguishes: functional hedging (genuine epistemic stance) vs. formulaic padding (using the phrase because the writer learned it, not to convey uncertainty).

**PMC Hedging Study 2021** (already cited in style.js Loop 33 impersonalPenalty): the same paper covers these "noting" frames as a separate category from the "it is said/believed" frames already penalized.

The existing `FORMULAIC_BUNDLES` in `vocabulary.js` covers: "it is important to", "it is necessary to", "there is no doubt", "we can see that/from", "it can be seen that", "it is obvious that", "it goes without saying".

**Not yet covered**: "it is important to note", "it should be noted that", "it is worth noting that", "it is essential to note", "it is crucial to note", "needless to say" (this last one IS in boosterPenalty).

### Mechanism
Add 4 specific noting/framing phrases to the existing `FORMULAIC_BUNDLES` array in vocabulary.js. These are a more specific subset of the academic hedging family and complement (not duplicate) the impersonalPenalty in style.js (which targets "it is said/believed" — passive epistemic stance, not "noting" frames).

### Code Sketch (vocabulary.js — expand FORMULAIC_BUNDLES)
```js
// Add to FORMULAIC_BUNDLES array (after Loop 27 entries):
/\bit is important to note\b/gi,
/\bit should be noted that\b/gi,
/\bit is worth noting that\b/gi,
/\bit is essential to\b/gi,     // "it is essential to note/understand/consider"
```

Same penalty trigger logic already in place: `maxSingleBundle >= 3 ? 0.04 : totalBundleHits >= 5 ? 0.02 : 0`

### Frequency Estimate
- Chinese L1 Score-3/4 essays: ~25–35% contain at least one of these; ~10% use 3+
- Native writers: "it should be noted that" used occasionally (≤1 per essay) — threshold of 3 same-phrase prevents FP
- Expected impact: minor (-0.02 on a subset of essays), but closes a lexical sophistication gap

### FP Risk
LOW. Same threshold logic as existing FORMULAIC_BUNDLES which has been stable. "It is important to note" once is not penalized. Only 3+ uses of same phrase OR 5+ total bundles fires.

### Priority: LOW-MEDIUM
Small marginal gain. Worth doing in <10 minutes because it's literally adding 4 lines to an existing array, but the calibration impact is small.
