## L57 (MEDIUM): development.js — 2-3 digit numbers + measurement units in numericEvidenceBonus

### What's missing

Current `numericEvidenceBonus` only counts:
- Percentages (`\d+%`)
- Years (`\b(19|20)\d{2}\b`)
- Large numbers (`\b\d{4,}\b` or comma-formatted)

The S4 calibration essay has "employees who previously spent **90 minutes** commuting". This is a specific statistic but not captured because 90 is a 2-digit plain number.

### Proposed pattern

```javascript
const MEASURED_NUM_RE = /\b\d{2,3}\s+(minutes|hours|days|weeks|months|years|percent|dollars|students|workers|employees|people|countries|miles|kilometers|meters|cm|kg|lb|items|points|times|more)\b/gi
const measuredNums = (text.match(MEASURED_NUM_RE) || []).length
numericSignals += measuredNums >= 1 ? 1 : 0
```

### FP analysis

- "90 minutes commuting" → always a real statistic in essay context
- "5 reasons" → not matched (unit must be a measurement, not a counting word)
- "a few times" → not matched (no leading integer for "times")
- FP: ~0% — requiring both a 2-3 digit number AND a measurement unit eliminates noise

## L58 (LOW): development.js — result-clause warrant (`which allows/enables/results in`)

### What's missing

S4 essay has: "eliminating daily commutes, **which allows** them to focus more on their actual responsibilities." This "action + which allows + consequence" structure is a sophistication marker per Toulmin (1958) argument theory — the writer has embedded the warrant within the claim.

Current `warrantedClaimBonus` requires THESIS_RE (I believe/I think) + CAUSAL_RE (because/since) in the SAME sentence. This misses embedded-consequence clauses.

### Proposed addition

```javascript
const RESULT_CLAUSE_RE = /,\s+which\s+(allows?|enables?|means?|results? in|leads? to|helps?|makes? it)/i
const resultClauseCount = sentences.filter(s => RESULT_CLAUSE_RE.test(s)).length
// Add to warrantedBonus: if resultClauseCount ≥ 1, +0.02
```

### FP analysis

- ", which allows" → always a result clause showing cause-consequence
- ", which means" → academic result clause ✓
- FP: ~0%

### Impact

- S4: +0.02 development → overall +0.028 → 4.27 → 4.3 (if L57 already in)
- Combined L57+L58 S4 estimate: 4.3 (gap to S5: 0.60)
- Calibration still passes
