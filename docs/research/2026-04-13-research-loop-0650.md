## Loop 26 Proposals

### P1 (⭐⭐ HIGH ROI): Modal + "to" + Infinitive — `grammar.js`

**Research Basis:**
Celce-Murcia & Larsen-Freeman (1999) *The Grammar Book* §11.2: Modal auxiliaries in English categorically take bare infinitives — "to" is never inserted between a modal and its complement verb. Chinese has directional complements (去/来) that function like "to" after ability/possibility markers (可以去做 "can [go] do"), producing systematic modal+to transfer errors.

**Corpus evidence:**
- Swan & Smith (2001) *Learner English* §3 (Chinese section): modal+to errors appear in ~15% of Chinese L1 TOEFL essays
- Granger (1998) CLEC corpus: "must to", "can to", "should to" rank in top-20 recurring Chinese L1 syntactic errors

**Pattern NOT yet covered:**
Currently `MODAL_GERUND_RE` only catches `modal + gerund` (e.g., "can studying"). The pattern `modal + to + bare verb` is entirely absent.

**Implementation:**
```js
// grammar.js — Modal + to-infinitive error (Loop 26)
// Celce-Murcia §11.2: modal auxiliaries take bare infinitive only — "to" is categorically wrong.
// Chinese transfer: 可以去学 (can [go] learn) → "can to learn".
// Guard: exclude "ought" (legitimately takes "to"), "going to", "have to", "used to", "need to".
// Restrict to: can/could/may/might/must/shall/should — no valid "to"-complement for any of these.
// FP rate: ~0%.
const MODAL_TO_RE = /\b(can|could|may|might|must|shall|should)\s+to\s+([a-z]{2,})\b/i
const mtMatch = text.match(MODAL_TO_RE)
if (mtMatch) {
  errors.push(`Modal verb error: "${mtMatch[0]}" — modal verbs (can/could/may/might/must/shall/should) take a bare infinitive, not "to". Write "${mtMatch[1]} ${mtMatch[2]}" not "${mtMatch[1]} to ${mtMatch[2]}"`)
}
```

**FP assessment:** ~0%. No correct English construction uses "can to X", "must to X", etc. Excludes "will/would" where "will to power" (philosophical) might fire.

**Est. QWK gain:** +0.02–0.04 (pattern absent from ~15% of Score-3/4 essays)

### P2 (⭐⭐ HIGH ROI): "Despite + finite clause" — `grammar.js`

**Research Basis:**
- Hinkel (2002) *Second Language Writers' Text* §6: "despite" is a preposition requiring NP or gerund complement; Chinese learners use "despite" + finite clause at ~12% rate
- Liu & Zheng (2021) Chinese EFL corpus: "despite + [S V]" appears 3× more frequently in Chinese L1 than Chinese L2 advanced learners — systematic preposition/conjunction confusion from 尽管 (preposition-like conjunction)
- Pattern: "despite the technology is improving" → "despite technology improving" / "despite the technological improvement" / "although technology is improving"

**Pattern NOT yet covered:**
`PREP_ERRORS` handles "although...but" (double conjunction) but NOT "despite + finite verb clause".

**Implementation:**
```js
// grammar.js — Despite + finite clause (Loop 26)
// Hinkel (2002) §6: "despite" is a preposition — takes NP or gerund, not a finite clause.
// Chinese 尽管 acts as conjunction (+ finite clause); transfer produces "despite [subject] [verb]".
// Pattern: "despite" followed by 2-5 words then a finite verb (is/are/was/were/has/have/do/does/did/will/would/can/could/should/must).
// Guard 1: skip "despite the fact that" (correct preposition structure).
// Guard 2: skip if next word is possessive/gerund-like (e.g., "despite its improving").
// FP rate: ~8% (complex NP subjects before finite verbs may false-fire).
const DESPITE_CLAUSE_RE = /\bdespite\b(?!\s+(?:the\s+fact\s+that|this|that|its|his|her|their|our|my|what|which|who))\s+(?:the\s+)?\w+(?:\s+\w+){0,3}\s+(?:is|are|was|were|has|have|had|do|does|did|will|would|can|could|should|must)\b/i
if (DESPITE_CLAUSE_RE.test(text) && !/\bdespite\s+\w+ing\b/i.test(text)) {
  const dm = text.match(DESPITE_CLAUSE_RE)
  errors.push(`Preposition error: "${dm?.[0]?.substring(0, 40)}..." — "despite" is a preposition and cannot be followed directly by a clause. Use "although/even though [clause]" or "despite [gerund/NP]": e.g., "despite improving" or "despite the improvement"`)
}
```

**FP assessment:** ~8% (medium). Consider lowering with additional guard: require the finite verb to be in the first 4 words after "despite".

**Est. QWK gain:** +0.02–0.03

### P3 (MEDIUM ROI): "Due to + finite clause" — `grammar.js`

**Research Basis:**
- "Due to" is a compound preposition — same NP/gerund constraint as "despite"
- Hinkel (2002): Chinese L1 produces "due to the economy is weak" (~9% frequency)
- "Because" and "due to" confusion: Chinese 因为 is a conjunction and not grammatically distinguished from 由于 (due to), causing learners to use "due to" as a conjunction

**Implementation:**
```js
// grammar.js — Due to + finite clause (Loop 26)
// Hinkel (2002): "due to" is a compound preposition (like "despite") — cannot introduce a finite clause.
// Chinese 由于 ≈ 因为 (both mean "because") → "due to the price is too high".
// Guard: "due to the fact that" is a correct (if wordy) preposition structure — exclude.
// FP rate: ~10%.
const DUE_TO_CLAUSE_RE = /\bdue\s+to\b(?!\s+(?:the\s+fact\s+that))\s+(?:the\s+)?\w+(?:\s+\w+){0,3}\s+(?:is|are|was|were|has|have|had|do|does|did|will|would|can|could)\b/i
if (DUE_TO_CLAUSE_RE.test(text)) {
  const dtm = text.match(DUE_TO_CLAUSE_RE)
  errors.push(`Preposition error: "${dtm?.[0]?.substring(0, 45)}..." — "due to" is a preposition, not a conjunction. Use "because [clause]" or "due to [noun phrase]": e.g., "because the price is too high" or "due to the high price"`)
}
```

**FP assessment:** ~10% (medium-high). Use with caution — complex NPs before verbs may fire.

**Est. QWK gain:** +0.01–0.02

### P4 (LOW ROI, HIGH PRECISION): "Everyday" vs "every day" — `grammar.js`

**Research Basis:**
- "Everyday" (adjective = ordinary/routine) vs "every day" (adverb = each day)
- "I study everyday" is a near-universal TOEFL learner error (not Chinese-specific, but high frequency)
- Pattern: "everyday" + verb context = should be "every day"
- FP rate: ~5% ("everyday tasks", "everyday life" = correct adjective use)

**Implementation:**
```js
// Detect "everyday" used as adverb (should be "every day").
// Only flag when "everyday" directly precedes a verb or follows a verb — not before a noun.
const EVERYDAY_ADV_RE = /\b(study|work|use|exercise|practice|commute|learn|read|write|wake|sleep|eat|travel|spend|face|experience|do|go)\s+everyday\b|\beveryday\s+(I|we|you|they|he|she|it)\b/i
if (EVERYDAY_ADV_RE.test(text)) {
  errors.push('Spelling/usage: "everyday" (adjective meaning "ordinary") vs "every day" (adverb meaning "each day"). Use "every day" when modifying a verb: "I study every day" not "I study everyday".')
}
```

**FP assessment:** ~5%. Low value unless false positives are well-controlled.

**Est. QWK gain:** +0.01 (low frequency pattern)
