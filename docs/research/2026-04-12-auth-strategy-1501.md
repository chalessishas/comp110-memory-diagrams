# Auth 方案研究：TOEFL SPA 跨设备分数同步

> Research Loop 自动触发 — 2026-04-12 15:01

## 背景

`scoreHistory.js` 已有版本化 localStorage 结构 `{ v: 1, data: [] }`，每条记录含 `type`、`promptIdx`、`score`、`date`。跨设备同步是用户回来后最可能的下一步决策。

---

## 四方案对比

| 维度 | Supabase 匿名 Auth | Supabase Magic Link | GitHub OAuth | Clerk |
|------|-------------------|--------------------|----|------|
| 新用户摩擦 | 零摩擦，自动创建 | 需输入邮箱 + 收邮件 | 需 GitHub 账号 | 可配置，但多一个 SDK |
| 迁移路径 | 直接：匿名登录 → 上传 localStorage JSON | 需先登录才能上传 | 同左 | 同左 |
| 升级路径 | 后续可 `linkIdentity()` 绑定邮箱/OAuth | 已是永久账户 | 已是永久账户 | 内置身份合并 |
| 免费额度 | 50K MAU | 50K MAU | 50K MAU | 10K MAU，超出 $0.02/MAU |
| 已有经验 | Signal-Map 已在用 | Signal-Map 已在用 | Signal-Map 已在用 | 无 |
| 实现工作量 | ~50 行 | ~80 行 | ~60 行 | ~100 行 |
| 学生用户适配 | 极好（无感知） | 一般（等邮件体验差） | 差 | 中等 |

---

## 推荐方案：Supabase Anonymous Auth → Magic Link 升级路径

**核心理由：**
1. 零摩擦：`signInAnonymously()` 用户无感知，打开即练习
2. 迁移最简：localStorage 数组直接 `upsert` 进 Supabase
3. 升级平滑：`linkIdentity()` 绑邮箱，数据不丢，user_id 不变
4. 已有 Supabase 经验（Signal-Map）
5. 免费额度 50K MAU（Clerk 只有 10K）

---

## 3 步实施路径（总计约 4 小时）

### Step 1 — 匿名 Auth 接入（~30 分钟）
```js
const { data: { user } } = await supabase.auth.signInAnonymously()
// 读 localStorage → upsert to scores 表
```

### Step 2 — 数据表设计
```sql
create table scores (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  type text, prompt_idx int, score int, date timestamptz,
  created_at timestamptz default now()
);
alter table scores enable row level security;
create policy "own" on scores for all using (auth.uid() = user_id);
```

### Step 3 — 邮箱绑定升级（~1 小时，用户主动触发）
```js
await supabase.auth.updateUser({ email: 'user@example.com' })
// 邮件确认后匿名账户升级为永久账户，数据全保留
```

## 风险提示

- **数据丢失风险**：用户清除浏览器数据会丢失匿名 session，需 UI 轻提示"绑定邮箱以跨设备同步"
- **僵尸账户积累**：需定期清理 90 天未登录匿名用户（Supabase 免费版 500MB 数据库限制）
- **简化替代**：若只需同一浏览器多标签同步，`BroadcastChannel` 无需任何后端，可将 Supabase 接入推后
