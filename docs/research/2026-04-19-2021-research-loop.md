## Open Gaps Identified

### 1. Charger Class Trait NOT Auto-Wired (HIGH)

The Charger Vanguard trait — "gain 1 DP when defeating an enemy" — is currently **per-operator opt-in**, not class-level dispatch:

- Only `fang.py` and `vigna.py` manually import and register `vanguard_charger_dp_on_kill`
- The 417 generated operators include Plume, Grani, Bagpipe, Reed, Wild Mane — all Chargers that won't auto-get this trait
- The P6.2 task from the prior research plan (`2026-04-19-1914`) is still open

**Fix path:** In `data/characters/registry.py`, after loading a generated/curated factory, check the unit's `subclass` field (or `archetype == RoleArchetype.VAN_CHARGER`). If it's a Charger, auto-attach the `charger_dp_on_kill` talent tag by calling the same `register_talent` hook Fang/Vigna use. Zero new infrastructure needed.

### 2. Vigna Crit Talent Still Deferred

Vigna's `vigna.py` explicitly defers the 10% crit × 1.5 multiplier talent due to RNG non-determinism in tests. This is acceptable, but a deterministic approach exists: add a `crit_chance` field to `UnitState` + `combat_system` seeds a `random.Random(42)` per-world. Tests that need determinism can use a fixed seed world. No urgency — low fidelity impact.

### 3. ATK Formula FLOOR Precision

The multi-FLOOR ATK formula from the research doc:
```
atk = floor(floor(atk_base × atk_stage) × (1 + sum_pct_buffs)) + atk_additive
```
Current `effective_stat()` appears to apply a single floor at the end. This may produce small rounding differences (~1-2 ATK) in high-buff scenarios. Not caught by tests because test assertions use `pytest.approx(rel=0.01)`. Not urgent for current feature work.

### 4. Arts Damage 0-HP Edge Case

Mentioned in chronicle entry `sonnet-0419AV` as a "low priority todo" — arts damage can theoretically bring HP to negative values since there's no 5% floor for arts. Current combat_system likely handles this correctly already (hp clamped at 0 before cleanup), but no dedicated test.
