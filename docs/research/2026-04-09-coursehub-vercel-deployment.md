## Vercel Function Region

**必须改为 `hnd1`（日本东京）。**

DashScope endpoint 在中国大陆。默认 `iad1`（美东）到 DashScope 延迟 500ms-2s；`hnd1` 到 DashScope 约 50-100ms，差 10 倍。

在 `vercel.json` 中设置：
```json
{
  "regions": ["hnd1"]
}
```

---

## Next.js 16 部署坑

**动态路由 params 必须 await（Next.js 15+ 要求）：**
```bash
grep -r "params\." src/app/api --include="*.ts" | grep -v "await"
```
扫描所有 `[id]` 路由，确认均已 `await params`。

**viewTransition: true**（已开启）：Next.js 16 中趋于稳定，但勿使用 `<Cache>` 组件（有已知 hydration bug）。

**Server Actions CORS**：项目全用 API Routes 无 Server Actions，无需配置 `allowedOrigins`。

## DashScope Serverless 风险

| 风险 | 状态 | 建议 |
|------|------|------|
| 网络延迟（海外调中国大陆）| 高 | 选 hnd1 region |
| AbortSignal 已注释 | ⚠️ 待修复 | 改为 `AbortSignal.timeout(55000)` 防止挂起 |
| 国际访问间歇性故障 | 低但存在 | 无法规避，监控 error rate |
| Vercel 计费按执行时间 | 无 AbortSignal 时风险高 | 同上，恢复 AbortSignal |
