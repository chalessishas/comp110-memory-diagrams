## 关键发现

### ⚠️ `streamObject` 在 ai@6 已标注 `@deprecated`

现有 `ai@6.0.145` 的源码中 `streamObject` 已被标注废弃。官方推荐路径：

```typescript
import { streamText, Output } from 'ai';

const { elementStream } = streamText({
  model: qwen("qwen3.5-plus"),
  output: Output.array({ element: chunkSchema }),
  prompt: "...",
});
```

**结论：直接用 `streamText + Output.array`，不用 `streamObject`。**

### DashScope 兼容性

`@ai-sdk/openai` 已在 `course-hub/src/lib/ai.ts` 正确配置 DashScope endpoint：

```typescript
const qwen = createOpenAI({
  baseURL: "https://dashscope.aliyuncs.com/compatible-mode/v1",
  apiKey: process.env.DASHSCOPE_API_KEY ?? "",
});
```

- SDK 发送 `response_format: { type: "json_schema", strict: true }` — DashScope 兼容接受
- DashScope 不支持 `tools + stream: true` 同时，但 `streamObject` 不用 tools，无影响
- **⚠️ Thinking mode 泄漏**：`qwen3.5-plus` 可能在 JSON 流中插入 `<think>...</think>` 块，破坏增量 JSON 解析。course-hub 已有 `stripThinkBlocks()` — 需要确认 streaming 路径也调用了它

### `array` + `elementStream` — 最适合 lesson chunks

| 属性 | 效果 |
|------|------|
| `partialObjectStream` | 每个 token 都发射，最后一个 chunk 可能含 undefined 字段 |
| `elementStream` | 只在元素完整且通过 Zod 验证后才发射，天然适合"一块块弹出"的 UX |

SDK 内部把数组包在 `{ elements: [...] }` 中发给 LLM，然后解包。开发者只看到 clean 的 element 序列。

**完整 Next.js API route 模式：**

```typescript
// app/api/generate-chunks/route.ts
import { streamText, Output } from 'ai';
import { z } from 'zod';

const chunkSchema = z.object({
  title: z.string(),
  content: z.string(),
  quiz: z.object({ question: z.string(), answer: z.string() }),
});

export async function POST(req: Request) {
  const { topic } = await req.json();
  const result = streamText({
    model: qwen("qwen3.5-plus"),
    output: Output.array({ element: chunkSchema }),
    prompt: `Generate 4 lesson chunks about ${topic}...`,
  });
  return result.toTextStreamResponse();
}
```

**Client side:**

```typescript
for await (const chunk of result.elementStream) {
  // chunk 是完整的 { title, content, quiz }
  setChunks(prev => [...prev, chunk]);
}
```

### 已知坑和限制

| 坑 | 影响 | 防护 |
|----|------|------|
| `strict: true` 默认开启，Qwen 可能忽略 | schema 不严格执行，但不会直接报错 | 用 `.nullable().default(null)` 替代 `.optional()` |
| Vercel 60s 超时 | 4 chunks × 15s/chunk ≈ 60s，恰好在边界 | 减少 chunks 数量或缩短 content 长度 |
| Thinking mode 泄漏破坏 JSON 流 | `streamText` 不会自动 strip | 需要在 `fullStream` 过滤 `reasoning` 类型事件 |
| `response_format + stream` 官方文档未明确说支持 | 可能只对某些模型版本生效 | 先用 `qwen-plus` 测试，再试 `qwen3.5-plus` |
| `AbortSignal.timeout()` 已被禁用 | 请求无超时保护 | 保持现状，或改用 `Promise.race` + `setTimeout` |
