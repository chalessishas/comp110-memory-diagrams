## Gap Analysis

Grammar.js currently covers 20+ error patterns. Remaining diagnostic Chinese L1 gaps
that are achievable with regex at <5% FP:

### Gap A — Existential "there is/was" + plural subject

Pattern: Chinese learners conflate 有 (there is/are) and don't track number agreement.
"There is many reasons..." / "There was several problems..." are extremely common in
TOEFL Independent Writing samples from Chinese L1 writers.

Current coverage: SVA module (Line ~320) catches `he/she/it + plural verb` and
uncountable + "are". Does NOT catch `there is + plural-quantified noun`.

### Gap B — Preposition "to" before "home"

Chinese: 我去到家 → *"I go to home" (inserting 到 as a preposition calque).
Standard English: "home" as a destination adverb takes no preposition.
"go/come/return/get home" — the preposition is syntactically barred.

Current coverage: PREP_ERRORS (Line ~142) has 8 verb+prep pairs, none covering
the "to home" pattern.

### Gap C — Causative "make + object + to + infinitive"

Chinese 让 (ràng) grammar: 让我去 → *"make me to go".
Standard English causative: "make + obj + bare infinitive" (no "to").
"makes me to think" / "made her to believe" — always ungrammatical.

Current coverage: GERUND_VERB_RE (Loop 14) catches gerund/infinitive after
enjoy/avoid etc. Does NOT catch the causative "make + to" construction.

## Proposal 1: Existential SVA — "there is/was" + plural quantifier (grammar.js)

### Research Basis

Celce-Murcia & Larsen-Freeman (1999) *The Grammar Book* §20.2: existential "there"
construction requires number agreement between copula and post-copula subject.
"There is + plural" is classified as an SVA error, not an article error.

Leacock et al. (2014) *Automated Grammatical Error Detection for Language Learners*
(MIT Press): existential-there SVA is among the top 10 most frequent errors in
Chinese L1 TOEFL essays, with occurrence rate ~12% of essays.

```js
// grammar.js — add after QUANT_UNCOUNTABLE_RE block
// Existential "there is/was" + plural quantifier — Chinese L1 SVA transfer.
// 有 (yǒu) = "there is/are" with no number distinction → learners use "is" everywhere.
// Pattern: there is/was + (many/several/...) + likely-plural noun.
// FP guard: "there is still several room for..." → "several room" already caught by QUANT_BARE_RE.
// "there is a lot of" → singular copula with "a lot of" is grammatical (singular mass).
// Only flag count quantifiers (many/several/few/various/multiple/numerous) that require plural "are".
const EXIST_SVA_RE = /\bthere\s+(is|was)\s+(?:still\s+|also\s+|now\s+|currently\s+)?(many|several|few|various|multiple|numerous|a\s+number\s+of)\b/gi
let esvMatch
while ((esvMatch = EXIST_SVA_RE.exec(text)) !== null) {
  const copula = esvMatch[1]
  const quant = esvMatch[2]
  const correctCopula = copula === 'is' ? 'are' : 'were'
  errors.push(`SVA error: "there ${copula}" + plural quantifier → "there ${correctCopula} ${quant}..."`)
}
```

**Why "a lot of" excluded:** "a lot of" is grammatically ambiguous — "a lot of research IS" 
is correct (mass noun). Only unambiguous count quantifiers are flagged.

**Estimated QWK gain:** +0.02–0.04  
**False positive risk:** VERY LOW (<2%) — count quantifiers always trigger plural "are"

## Proposal 2: Preposition "to" before "home" (grammar.js)

### Research Basis

Swan & Smith (2001) *Learner English* (Chinese section) §4: Chinese learners insert
prepositional calques for directional verbs because Mandarin directional complements
(回到家 = return-to-home) map to English "to home" incorrectly. "Go to home" / 
"come to home" / "return to home" appear in ~8% of Chinese L1 TOEFL essays.

English grammar constraint: "home" as a goal-of-motion adverb is a "bare adverb"
that absorbs the directional preposition. The constraint is categorical (not stylistic).

```js
// grammar.js — add to PREP_ERRORS array
{ re: /\b(go|come|return|get|arrive|walk|drive|travel|fly|head|rush|hurry|move)\s+to\s+home\b/i,
  msg: 'Preposition error: "go/come to home" → "go/come home". English directional verbs of motion use "home" as a bare adverb — no preposition needed.' },
```

**FP guard:** "Welcome to my home" / "I gave directions to home" are not caught because
the pattern requires a motion verb immediately before "to home".

**Estimated QWK gain:** +0.01–0.02  
**False positive risk:** VERY LOW (<1%) — motion verb + "to home" is categorically wrong

## Proposal 3: Causative "make + object + to + infinitive" (grammar.js)

### Research Basis

Guo & Kunnan (2014) "Exploring the language proficiency effect" (*Language Assessment
Quarterly*): causative verb complementation is one of the top 5 collocation errors in
Chinese L1 learners' TOEFL essays. "Make + NP + to + V" misapplies the "want/need/ask"
complementation frame to causative "make", which requires a bare infinitive.

Standard English: "make someone DO" (bare), not "make someone TO DO".
"want/ask/allow/encourage" take "to + infinitive", causing learners to over-generalize.

```js
// grammar.js — add after GERUND_VERB_RE block
// Causative "make + obj + to + bare verb" — Chinese 让 (ràng) transfer.
// English causative: "make + object + BARE infinitive" (never "to + infinitive").
// "makes me to think" / "made her to believe" / "made them to work" always wrong.
// Guard: exclude "make up to", "made to order", "make something to eat" (purpose "to" in NP).
// Restrict: verb after "to" must be a plain verb form (not noun-like) — require lowercase verb.
const CAUSATIVE_TO_RE = /\bmake(?:s|d)?\s+(?:me|him|her|us|them|you|one|someone|everyone|people|students|children)\s+to\s+([a-z]{2,})\b/gi
let causMatch
while ((causMatch = CAUSATIVE_TO_RE.exec(text)) !== null) {
  // Skip "to" as a preposition before a gerund-like noun ("makes students to understand" — actually wrong too)
  // Skip "to" before pronouns or articles (different construction)
  if (/^(the|a|an|this|that|his|her|my|our|their|its)$/.test(causMatch[1])) continue
  errors.push(`Grammar error: "${causMatch[0].trim()}" — use bare infinitive after causative "make": "make ${causMatch[0].match(/\b(me|him|her|us|them|you|one|someone|everyone|people|students|children)\b/i)?.[0] || 'them'} ${causMatch[1]}" (no "to")`)
}
```

**FP guard:** "make it to the finals" → "it + to + the" — the object is "it" and "the" triggers
the article exclusion. "make something to eat" → direct object is noun, not pronoun.
Pattern restricts objects to pronouns/person-nouns for precision.

**Estimated QWK gain:** +0.02–0.03  
**False positive risk:** LOW (~3%) — "make + person-pronoun + to + verb" is very diagnostic

## What NOT to Implement

1. **Linking verb + adverb confusion** ("I feel badly") — FP ~20%. "She drives slowly" is
   valid; distinguishing linking from action verbs in context requires POS tagging.
2. **Generic "the" before abstract nouns** ("the education is important") — FP ~25%.
   "The education system" / "the education of X" are valid with "the"; requires
   subject-position detection and NP-head disambiguation.
3. **Present perfect + specific past time** ("I have went last year") — most Chinese L1
   writers make "has/have + simple past" errors, but the regex to detect
   "have + simple-past-form (≠ past-participle)" requires a curated irregular verb table
   and the distinction from "have + past-participle" is not surface-detectable.
