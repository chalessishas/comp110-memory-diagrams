## Pattern Candidates for Loop 34

### Pattern 1 — `result to` (Wrong preposition after "result")

**Error example:** "This will result to many problems."
**Correct form:** "This will result in many problems."
**Chinese source:** Chinese 导致 (dǎozhì) is a transitive verb with no preposition needed; when learners reach for "result", they randomly pick "to" (influenced by "lead to") instead of the fixed collocation "result in". The error pattern "result to" appears in CLEC-based corpus studies on preposition collocations.

**Regex sketch:**
```js
{ re: /\bresult(?:s|ed|ing)?\s+to\b/i,
  msg: 'Preposition error: "result to" → "result in". The verb "result" always collocates with "in": "result in problems/benefits/changes". ("Lead to" uses "to" — but "result" uses "in".)' }
```

**Frequency estimate:** ~4-6% of Chinese L1 TOEFL essays based on CLEC preposition collocation data (City University HK ELSS corpus; Laufer & Waldman 2011 verb+prep list)
**FP risk:** VERY LOW — "result to" is categorically non-standard in all English registers. The only near-miss is "the result to" (noun phrase), but that requires "the" preceding it, which the regex would not match.
**Additional note:** "result from" (passive causal direction) is correct and must NOT be flagged. The regex is anchored on "result + to" specifically, so "result from" is safe.
**Corpus source:** CLEC preposition error studies; City University HK ELSS collocation resource (same source that validated `emphasize on`, `discuss about`)

### Pattern 2 — `make + adjective` without object (Missing obligatory object)

**Error example:** "Technology makes convenient for people." / "This policy makes difficult to achieve."
**Correct form:** "Technology makes life convenient." / "This policy makes it difficult to achieve."
**Chinese source:** Chinese causative structure 使+adj (使方便 = make convenient) transfers directly without the obligatory English object. Grammar.js already has Loop 33 covering `makes convenient` / `make possible` but only for the fixed high-frequency set. The broader pattern — `make/makes/made + adjective directly` — is not covered for the full adjective set seen in TOEFL writing.

**Regex sketch:**
```js
// "make/makes/made" immediately followed by adjective (no intervening noun/pronoun)
// Exclude: "make it/them/this/that/things/life/the situation + adj" (correct forms)
// Exclude: "makes sense" / "makes progress" (fossilized idioms)
const MAKE_ADJ_RE = /\b(make|makes|made)\s+(convenient|difficult|easier|harder|better|worse|possible|impossible|necessary|important|clear|obvious|meaningful|effective|efficient|useful|valuable|dangerous|harmful|beneficial|equal|fair|unfair|safe|comfortable|enjoyable|interesting|attractive|available|sustainable|feasible|practical)\b(?!\s+(?:to\s+\w+|for\s+\w+))/gi
// FP note: "make clear" in "let me make clear that..." IS idiomatic — needs guard
// Safer guard: fire only when preceded by subject + make (not "to make adj" infinitive clause)
```

**Frequency estimate:** ~8-12% of Chinese L1 essays based on CLEC verb-complement data. Already flagged by existing Loop 33 patterns for `makes convenient / make possible / makes difficult` but only for that 3-item whitelist. Full adjective expansion adds ~6% more coverage.
**FP risk:** LOW-MEDIUM. Key exceptions: `make clear` (idiomatic declarative), `make sense`, `make do`. These are all frozen idioms and can be excluded explicitly. The issue is "to make something adj" infinitive clauses — need to exclude if the trigger is inside a "to + make" infinitive.
**Corpus source:** Laufer & Waldman (2011) causative verb analysis; grammar.js Loop 33 already demonstrates the pattern is real and detectable.

### Pattern 3 — `succeed to do` / `insist to do` (Verb + `to` infinitive instead of gerund)

**Error examples:**
- "She finally succeeded to get the job." → "She finally succeeded in getting the job."
- "He insisted to leave early." → "He insisted on leaving early."
- "They devoted to improving education." → "They devoted themselves to improving education." / "They were devoted to improving education."

**Chinese source:** Chinese 成功 (succeed), 坚持 (insist), 致力于 (devote to) are all followed by bare verbs. Learners either default to "to + infinitive" (over-generalizing the infinitive complement pattern) or use a bare preposition without the required gerund. This is a gerund-after-preposition error identical in mechanism to the already-implemented `look forward to do` (Loop 23) and `for the purpose of + bare inf` (Loop 30), but for a different verb set.

**Regex sketch:**
```js
// Pattern A: "succeed to + bare verb" (should be "succeed in + gerund")
{ re: /\bsucceed(?:s|ed|ing)?\s+to\s+(?!the\b|a\b|an\b)(\w+)\b(?!ing\b)/i,
  msg: 'Preposition error: "succeed to [verb]" → "succeed in [verb]ing". The verb "succeed" collocates with "in" + gerund: "succeed in achieving / succeed in solving".' }

// Pattern B: "insist to + bare verb" (should be "insist on + gerund")
{ re: /\binsist(?:s|ed|ing)?\s+to\s+(?!the\b|a\b|an\b)(\w+)\b(?!ing\b)/i,
  msg: 'Preposition error: "insist to [verb]" → "insist on [verb]ing". The verb "insist" collocates with "on" + gerund: "insist on doing / insist on staying".' }

// Pattern C: "devote to + bare verb" (should be "devote to + gerund")
// "devote" requires gerund after "to" — same mechanism as "look forward to doing"
{ re: /\bdevote(?:s|d|ing)?\s+to\s+(?!the\b|a\b|an\b|their\b|his\b|her\b|my\b|our\b)(\w+)\b(?!ing\b)/i,
  msg: 'Gerund error: "devote to [bare verb]" → "devote to [verb]ing". "To" in "devote to" is a preposition, not an infinitive marker: "devote to improving" not "devote to improve".' }
```

**Frequency estimate:**
- `succeed to do`: ~4% in CLEC (verb complement error category; Xia 2012 infinitive study)
- `insist to do`: ~3-5% — documented in multiple CLEC-based studies on verb + preposition errors
- `devote to do`: ~3% — "devote" appears frequently in TOEFL Independent Writing ("devoted to solving social problems")
**FP risk:** VERY LOW for `succeed to` and `insist to`. The preposition "to" after "succeed/insist" followed by a bare verb is categorically wrong. Only risk: "succeed to the throne" (monarchy sense) — but this is irrelevant to TOEFL Independent Writing topics.
**Corpus source:** Xia (2012) CLEC-based infinitive error study (ccsenet.org, ELT v5n6p154); City University HK ELSS verb+preposition list; Laufer & Waldman (2011) Chinese EFL verb complement errors

### Pattern 4 — `the + uncountable noun (society/education/technology) in subject position` + present tense (Generic Article Overuse)

**Error example:** "The society should protect the environment." / "The technology has changed people's lives." / "The education is important for development."
**Correct form:** "Society should protect the environment." / "Technology has changed people's lives." / "Education is important for development."
**Chinese source:** Chinese uses 社会/教育/技术 as bare nouns with no article. When learners learn "the" for specific references, they over-apply it to generic mass/abstract nouns at the start of sentences. This is the opposite problem from article omission — it is article over-insertion on generic abstract nouns. This manifests distinctively in TOEFL Independent Writing because topics consistently invoke these exact nouns.

**Regex sketch:**
```js
// "The [abstract/uncountable TOEFL noun]" in subject position of a new sentence
// Trigger: sentence-start or after period — "The society/The education/The technology..."
// Verb requirement: present tense (is/are/has/have/should/can/must/plays/affects/helps)
// This restricts to generic statements, not specific references ("The technology used in this study...")
const THE_GENERIC_TOEFL = /(?:^|[.!?]\s+)The\s+(society|education|technology|environment|economy|culture|government|science|nature|poverty|globalization|communication|transportation|healthcare|media|youth|elderly)\s+(?:is|are|was|were|has|have|should|can|must|will|would|plays?|affects?|helps?|influences?|changes?|develops?|improves?|benefits?|provides?|requires?|needs?|matters?|contributes?)/gm
```

**Frequency estimate:** Already partially covered by Loop 17-18 patterns (grammar.js line 573: `THE_GENERIC_SAFE`). Check current coverage before implementing to avoid duplicate.
**IMPORTANT: Pre-implementation check required.** grammar.js line 573 already has `THE_GENERIC_SAFE` covering `technology/society/education/environment/economy/culture/science/nature/poverty`. Loop 34 should verify whether the existing pattern fires correctly before adding new nouns (`globalization/communication/healthcare/media/youth`). If the existing pattern is already catching these, this becomes a lower-priority expansion rather than a new pattern.
**FP risk:** MEDIUM. "The society she lived in" (specific reference) is correct — the verb list guard handles most of these cases. "The government has decided" is often specific, not generic. Restrict to a tight verb whitelist: is/are/should/can/must/plays/affects/influences/needs.
**Corpus source:** CLEC article error studies; Chinese L1 TOEFL corpus (Frontiers in Psychology 2021, fpsyg.2021.765983)
