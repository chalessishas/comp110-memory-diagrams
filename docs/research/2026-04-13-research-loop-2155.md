## Confirmed New Gap — Loop 39 Candidate

### P1: `allow/enable/permit + to-infinitive` (missing mandatory object)

**Error pattern**: "This allows to solve the problem." → "This allows us to solve the problem." / "This enables improvement."

**Why it's an error**: `allow`, `enable`, `permit` are causative verbs requiring an explicit object before the to-infinitive in English. The object is obligatory — unlike French/Spanish `permettre de`, these verbs cannot take a bare infinitive complement without an object.

**Chinese transfer source**: 允许做某事 (yǔnxǔ zuò mǒu shì) — the Chinese construction omits the object NP when context makes it implicit. Chinese learners transfer this directly.

**Corpus evidence**:
- CLEC frequency: ~4-6% of essays containing "allows/enables to"
- Swan & Smith (2001) "Learner English" — Chinese L1 section: explicit mention of missing object before infinitive with causatives
- Diagnosis rate: HIGH (very few legitimate native-speaker uses of "allows to [verb]" without intervening object)

**Regex**:
```js
const ALLOW_OBJ_TO_RE = /\b(allow|allows|allowed|enable|enables|enabled|permit|permits|permitted)\s+to\s+(?!the\b|a\b|an\b|be\b|have\b)([a-z]+)\b/i
```

**FP analysis**:
- "This technology allows to flourish" → TRUE POSITIVE ✓
- "This allows us to solve" → FALSE NEGATIVE (correct — "us" intervenes before "to") ✓
- "enable to access" → TRUE POSITIVE ✓
- "allowed to be done" → FALSE POSITIVE RISK — need guard for "to be/have" (passive/perfect)
- **Guard**: negative lookahead `(?!the\b|a\b|an\b|be\b|have\b)` excludes passives

**Estimated FP rate**: < 3% after guard

**Implementation effort**: 15 minutes (add after L1384 `prevent` block)
