## 1. Vite SPA 部署选项对比

| 平台 | 免费额度 | 优势 | 适合场景 |
|------|---------|------|---------|
| **Vercel** | 慷慨，无带宽上限 | 自动检测 Vite，SPA rewrite 自动配置，CLI 一键部署 | 最推荐：零配置 |
| **Netlify** | 慷慨 | Git 集成简单，output dir 设置直观 | 个人项目首选备选 |
| **Cloudflare Pages** | **无限带宽/请求** | 静态资源分发最优 CDN，免费层最宽松 | 高流量或全球用户 |
| **GitHub Pages** | 免费 | 零成本，仓库直出 | 最简单但功能弱 |

**对 TOEFL 项目的建议**：Vercel 是最快路径。
- `npm run build` → `vercel --prod` 即可
- SPA routing（React Router）需要 `vercel.json` rewrites 或 Vercel 自动处理
- 无后端依赖，纯静态 SPA，零运行成本

**Sources**: vite.dev/guide/static-deploy, digitalapplied.com/blog/vercel-vs-netlify comparison

## 3. React localStorage 版本控制 / 迁移模式

**核心策略**：在存储数据中加入 `version` 字段，schema 破坏性变更时 bump 版本，顺序执行迁移函数。

```js
// 推荐模式（适合 TOEFL 项目）
const CURRENT_VERSION = 2;
const migrate = (raw) => {
  if (!raw.version || raw.version < 2) {
    // v1 → v2: add breakdown field to history entries
    raw.entries = raw.entries.map(e => ({ breakdown: null, ...e }));
    raw.version = 2;
  }
  return raw;
};
```

**对 TOEFL 项目的风险评估**：
- 目前有 5 个独立 localStorage keys，各自结构不同
- `toefl-completion-history` 格式若变更（如 scoreHistory.js 加字段），老用户数据静默失效
- 当前无 `version` 字段 — **建议在下一次 schema 变更前加入**

**Available libs**: `versioned-storage`（轻量），Zustand persist middleware（若迁移到状态管理）

**Sources**: janmonschke.com/simple-frontend-data-migration, zustand docs, github.com/CatChen/versioned-storage
