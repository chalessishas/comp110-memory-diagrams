## Current Scorer Gap Analysis

After reviewing the live scorer modules, three concrete gaps remain:

---

### Gap 1: Vocabulary scorer doesn't distinguish academic words from obscure rare words

**File:** `vocabulary.js`
**Problem:** The `rareScore` is based on words NOT in `commonWords`. This rewards any uncommon word equally — "ubiquitous" and "bamboozle" both count as "rare". ETS specifically values Academic Word List (AWL) words: words used across academic disciplines that signal formal register and scholarly vocabulary.

**Concrete fix (est. 30min):**
Add a small set (~200 core AWL words) as a module-level constant. For each AWL word found in the essay, add +0.05 bonus (capped at +0.2) to the vocabulary score. This directly rewards the kind of vocabulary TOEFL rewards.

```js
const AWL_CORE = new Set([
  'analyze','approach','area','assessment','assume','authority','available',
  'benefit','concept','consistent','context','contract','create','data',
  'define','distribute','economic','environment','establish','evidence',
  'export','factors','financial','formula','function','identify','income',
  'indicate','individual','interpret','involve','issue','labor','legal',
  'major','method','occur','percent','period','policy','principle',
  'procedure','process','require','research','respond','role','section',
  'significant','similar','source','specific','structure','theory','vary',
  // ... ~150 more
])
```

### Gap 2: Organization scorer doesn't penalize misplaced discourse markers

**File:** `organization.js`
**Problem:** `countUniqueMarkers()` counts ALL markers regardless of placement. Using "in conclusion" in the first sentence, or "firstly" in the conclusion, scores just as well as correct placement. ETS rubrics specifically assess whether "markers form a logical sequence".

**Concrete fix (est. 1h):**
Add a placement check: if "in conclusion / to summarize / overall / in summary" appear in the LAST paragraph → bonus. If they appear in the FIRST paragraph → small penalty. This requires splitting text into paragraphs, taking the last one, and checking it for closing markers.

```js
const CLOSING_MARKERS = ['in conclusion','to summarize','overall','in summary','therefore']
const lastParagraph = paragraphs[paragraphs.length - 1]?.toLowerCase() || ''
const closerInLast = CLOSING_MARKERS.some(m => lastParagraph.includes(m))
const closerInFirst = CLOSING_MARKERS.some(m => paragraphs[0]?.toLowerCase().includes(m))
const placementBonus = closerInLast ? 0.1 : (closerInFirst ? -0.1 : 0)
```

---

### Gap 3: Development scorer doesn't detect claim repetition

**File:** `development.js`
**Problem:** An essay can score well by having a thesis, 2 reason markers, and 2 detail markers — even if it restates the same claim 4 times in different words. ETS explicitly penalizes "inadequate elaboration" which includes circular restatement without new information.

**Concrete fix (est. 45min):**
After splitting into sentences, check whether 3+ sentences have similar "claim fingerprints" — sentences that begin with "I believe / I think / I agree" without an example or reason marker in that same sentence. Each cluster of 3+ bare-claim sentences (no detail marker in the same sentence) = repetition flag, penalize -0.1 from argScore.

## Listening Module: Tradeoff Analysis (for user decision)

The user needs to decide between:

| Option | Pros | Cons |
|--------|------|------|
| Web Speech API (SpeechSynthesis) | Zero cost, works offline, no setup | Robotic voice, variable quality across browsers, limited to system TTS voices |
| ElevenLabs API | Near-human quality, consistent | $0.18/1k chars, requires API key, 15 TOEFL passages ≈ ~$2-5 total |
| Pre-recorded MP3s | Most authentic TOEFL-like | Storage (~5-10MB), must find or create audio files, copyright concern for real TOEFL |
| AWS Polly / Google TTS | High quality, cheap ($0.004/1k chars) | Requires AWS/GCP account setup |

**Recommendation:** Start with Web Speech API — it works immediately with no cost/setup, and can be replaced later if quality is insufficient. The existing TOEFL listening format (35 minutes, ~3-5 conversations + lectures) is well-defined; a simple TTS wrapper for the text content is a sufficient MVP.
