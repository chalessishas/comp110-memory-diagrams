## 1. SM-2 SRS Algorithm for Vocabulary

**Package**: [`supermemo`](https://www.npmjs.com/package/supermemo) — zero deps, 1.5 kB, ESM/CJS

**Algorithm summary**: Tracks `{ interval, repetition, efactor }` per item. Grade 0–5 after each review. Grade ≥ 3 = remembered; < 3 = reset to day 1. Interval grows geometrically (day 1 → day 6 → day 6 × efactor...).

```js
import { supermemo } from 'supermemo';
const next = supermemo({ interval: 0, repetition: 0, efactor: 2.5 }, grade);
// Store next + (Date.now() + next.interval * 86400000) as dueDate
```

**Integration priority**:
1. **Notebook (flashcard) — HIGH value**: Words accumulate over weeks, spacing matters. Store `{ interval, repetition, efactor, dueDate }` per word. Show "due today: N" badge. ~40 lines of new logic.
2. **BuildSentence — LOW marginal value**: Current wrong-pool captures ~80% of SRS benefit. Keep existing logic; SM-2 adds complexity for a short-session drill format.

**Gotcha**: `dueDate` is clock-dependent; localStorage has no cross-device sync. If user switches device, SRS state is lost.

## 2. PWA Support (vite-plugin-pwa)

**Setup** — under 20 lines:

```bash
npm install -D vite-plugin-pwa
```

```js
// vite.config.js
import { VitePWA } from 'vite-plugin-pwa';
VitePWA({
  registerType: 'prompt',   // NOT autoUpdate — prevents mid-test reloads
  manifest: {
    name: 'TOEFL Practice',
    short_name: 'TOEFL',
    theme_color: '#00695c',
    icons: [
      { src: 'pwa-192x192.png', sizes: '192x192', type: 'image/png' },
      { src: 'pwa-512x512.png', sizes: '512x512', type: 'image/png', purpose: 'any maskable' }
    ]
  }
})
```

**Gotchas**:
- Both 192×192 and 512×512 icons required in `public/` (Chrome refuses install without both)
- Use `pwa-asset-generator` to generate icons from source SVG
- `english-words.js` (2.7 MB) should be excluded from SW pre-cache via `workbox.globIgnores` — offline penalty too high
- iOS Safari: `beforeinstallprompt` doesn't fire, needs `apple-touch-icon` meta tag; install is manual
- HTTPS required — PWA install prompt never fires on plain HTTP

**ROI**: 2–3 hours for significant mobile UX improvement (offline + Add to Home Screen). High value for a daily-use study app.
